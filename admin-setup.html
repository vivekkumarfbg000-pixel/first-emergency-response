SQL Activation Helper

Supabase free-tier accounts limit SMTP confirmation emails to 3 per hour. Copy and run this script in your Supabase SQL Editor to confirm the account and assign administrative role permissions. 

-- 1. Grant execute permissions on public functions to resolve permissions bugs
GRANT EXECUTE ON ALL FUNCTIONS IN SCHEMA public TO public, anon, authenticated;
GRANT EXECUTE ON ALL FUNCTIONS IN SCHEMA public TO postgres, service_role;

-- 2. Force confirm the email of the admin account in auth.users
UPDATE auth.users
SET email_confirmed_at = now(),
    updated_at = now()
WHERE email = 'firstemergencyresponse4@gmail.com' 
   OR email = 'firstemergencycall@gmail.com' 
   OR email = 'firstemergencycall4@gmail.com';

-- 3. Ensure the 'admin' role is assigned in public.user_roles
INSERT INTO public.user_roles (user_id, role)
SELECT id, 'admin'
FROM auth.users
WHERE email IN (
  'firstemergencyresponse4@gmail.com',
  'firstemergencycall@gmail.com',
  'firstemergencycall4@gmail.com'
)
ON CONFLICT (user_id) 
DO UPDATE SET role = 'admin';
  1. Open the Supabase Dashboard.
  2. Select your active project, then click the SQL Editor in the left sidebar.
  3. Click New Query, paste the copied SQL block, and click Run.
